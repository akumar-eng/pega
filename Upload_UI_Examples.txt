===============================================================================
UI FORMATTING DETAILS
===============================================================================

The upload modal displays information with the following styling:

1. Modal Header: "Upload Attachments" (large, bold)
2. File Input: Standard file picker button
3. "Supported File Types:" (14px, bold, secondary color)
4. Main description text (12px, normal weight, secondary color, line-height: 1.4)
5. "Accepted Extensions:" (11px, bold, secondary color)
6. Extension lists (10px, gray color #666)
   - Category names are bold
   - Extensions are comma-separated
   - Scrollable if content exceeds 100px height

===============================================================================
DEDUPLICATION LOGIC
===============================================================================

The component implements smart deduplication to avoid showing the same 
extension in multiple categories:

Priority Rules:
- .csv, .tsv → Prioritized under "Spreadsheets" 
- .xml, .yaml, .yml → Prioritized under "Data"
- Other conflicts → First category wins

This ensures each extension appears only once, providing a clean user experience.

===============================================================================
TECHNICAL IMPLEMENTATION NOTES
===============================================================================

1. The component uses two helper functions:
   - getSupportedFileTypesDescription(): Returns human-readable descriptions
   - getDetailedFileExtensions(): Returns organized extension lists

2. Deduplication is handled automatically to prevent confusion

3. Categories are case-insensitive and whitespace-trimmed

4. The UI is responsive and scrollable for large lists

5. All newly added file types (TIFF, CSV, XML, etc.) are properly supported
